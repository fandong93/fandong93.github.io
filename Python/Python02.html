一、面向对象相关术语

  • 类：用来描述具有相同的属性和方法的对象的集合。它定义了该集合中每个对象所 共有的 属性和方法。对象是类的实例。

  • 对象：通过类创建出的实例。对象包括两个数据成员（类变量和实例变量）和方法。

  • 属性：类中的所有变量都被称为属性。

  • 方法：类中的所有函数通常称为方法。不过，和函数所有不同的是，类方法至少要包含一个 self 参数。

  • 方法重写：如果从父类继承的方法不能满足子类的需求，可以对其进行改写，这个过程叫方法的覆盖（override），也称为方法的重写。

  • 方法重载：类中支持多个同名函数，但参数的个数或类型要不同，在调用的时候，解释器会根据参数的个数或者类型，调用相应的函数.

  • 封装：将类的某些部分（属性、方法）隐藏起来，具有一定的保护作用，隐藏对象的属性和方法实现细节，仅对外提供公共的访问方式。

  • 继承：即一个派生类（derived class）继承基类（base class）的字段和方法。继承也允许把一个派生类的对象作为一个基类对象对待。

  • 多态：多态指的是一类事物有多种形态，（一个抽象类有多个子类，因而多态的概念依赖于继承）.

  • 实例化：创建一个类的实例，类的具体对象。

3.1 方法

3.1.1 构造方法

  • 当创建一个类的实例对象时，Python 解释器会自动调用这个类的构造方法。

  • 即使在类中不明确添加任何构造方法，Python 也会为类自动添加一个仅包含 self 参数的构造方法。

    • 仅包含 self 参数的 __init__() 构造方法是类的默认构造方法。

  • self 详解：

    • Python 只是规定，无论是构造方法还是实例方法，最少要包含一个参数，并没有规定该参数的具体名称。之所以将其命名为 self，只是程序员之间约定俗成的一种习惯，遵守这个约定，可以使我们编写的代码具有更好的可读性（大家一看到 self，就知道它的作用）。

    • 同一个类可以产生多个对象，当某个对象调用类中的方法时，该对象会将自身的引用作为一个参数自动地传给方法，类的方法会将第一个参数 self 用于接收传递来的引用。这样 Python 解释器就会知道要操作哪个对象的方法了。

    • 对于构造函数中的 self 参数，其代表的就是当前正在初始化的类对象。

3.1.4 类静态方法

  • 类静态方法类似于函数，和函数唯一的区别：静态方法定义在类这个空间（类命名空间）中，而函数则定义在程序所在的空间（全局命名空间）中。

  • 静态方法没有类似 self、cls 这样的特殊参数，因此 Python 解释器不会对它包含的参数做任何类或对象的绑定。也正因为如此，类的静态方法中无法调用任何类属性和类方法。

3.2 属性

  • 类变量（类属性）：定义在类体中且在方法体之外的变量被称为类变量（类属性），类变量在所有实例化对象中是共享的。

    • 类变量既可以使用类名调用：类名.变量名；也可以使用类的实例化对象调用：对象名.变量名。

    • 通过类名访问一个不存在的类变量时，就是为类和对象 动态增加类变量。

    • 类变量为所有实例化对象共有，通过类名修改类变量的值，会影响所有实例化对象。

    • 通过类的实例化对象对类变量进行赋值，其本质不是修改类变量的值，而是为该对象定义新的实例变量。

  • 实例变量（实例属性）：以 self.变量名 格式定义在方法体中的变量被称为实例变量，它只属于调用方法的对象。

    • 实例变量只能通过对象名访问，无法通过类名访问。

    • 当实例变量和类变量同名时，对象会调用实例变量，而不是类变量（类变量需要 类名.变量名 才可以访问）。

    • 通过对象修改实例变量的值，不会影响同类的其它实例化对象，更不会影响同名的类变量。

  • 局部变量：以 变量名 = 变量值 格式定义在方法体中的变量被称为局部变量。

    • 局部变量只能用于所在方法中，方法执行完成后，局部变量也会被销毁。

  • 全局变量：定义在全局命名空间内，所有类之外的变量被称为全局变量。

    • 不属于类的属性，但是可以被类的对象使用。

    • 当全局变量与局部变量同名时，优先访问局部变量（相当于新建了一个同名的局部变量）。

      • 可以使用 global 关键字来明确表示访问全局变量。

四、类的对象

4.1 类对象的创建

  • 类对象的创建又称为类的实例化，语法：类名(参数)。

    • 如果构造函数仅有一个 self 参数，我们无需为 self 传参。

    • 如果构造函数有多个参数时，我们需要为构造函数传递参数，否则抛出 TypeError 异常。

4.2 类对象的使用

  • 使用已创建好的类对象，通过 类对象.变量名 的格式访问或修改类中的实例变量。

    • 当类对象访问一个不存在的实例变量并赋值时，就是为已创建好的对象 动态增加实例变量。

      • 每个实例之间新增的实例变量是互不干扰的。

    • 可以使用 del 关键字 动态删除实例变量，格式：del 类对象.变量名。

  • 使用已创建好的类对象，通过 类对象.方法名(参数) 的格式调用类中的方法。

    • 另一种方法调用的方式：将 类对象.方法名 赋值给一个 变量，然后在变量后添加 (参数) 来调用，即 变量(参数)。

五、封装

  • 封装，即隐藏对象的属性和方法，这样在使用此类时，将无法直接以 “类对象.属性名”（或 “类对象.方法名(参数)”）的形式调用这些属性（或方法），而只能使用未隐藏的类方法间接操作这些隐藏的属性和方法。

  • Python 类中的变量或方法，只存在公有（public）和私有（private）两种访问权限，这 2 种权限的区别如下：

    • public：公有权限的变量或方法，在类的外部、类内部以及子类中，都可以正常访问。

    • private：私有权限的变量或方法，只能在本类内部使用，类的外部以及子类都无法使用。

  • Python 并没有提供 public、private 这些权限修饰符。为了实现类的封装，Python 采取了下面的方法：

    • 默认情况下，Python 类中的变量或方法都是公有（public）的，它们的名称前都没有下划线（__）。

    • 如果类中的变量或方法，其名称以 双下划线 “__” 开头，则该变量（方法）为私有变量（私有方法），“__” 等同于 private。

  • 以 单下划线 “_” 开头的变量或方法（例：_age、_info(self)）通常也被视为私有变量或私有方法，但这仅仅只是一个约定，Python 解析器并不会真的阻止用户去访问这个变量或方法，即以 “_” 开头的变量或方法也能通过类对象正常访问。

  • 封装原理：Python 封装仅仅是一种变形操作，即将类中所有双下划线开头的变量名或方法名（例：Test 类中 __age、__info(self)），自动变形成：_类名__变量名 或 _类名__方法名 的形式（例：_Test__age、_Test__info(self)）。

    • 这种机制并没有真正意义上限制我们从外部直接访问属性，知道了类名和变量名（方法名）就可以拼出名字：_类名__变量名，然后进行访问了。

六、继承
