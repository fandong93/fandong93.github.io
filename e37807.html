\quad 和其它编程语言一样，Python 也具有操作文件（I/O）的能力，例：打开文件、读取和追加数据、插入和删除数据、关闭文件、删除文件等。除了提供文件操作基本的函数之外，Python 还提供了很多模块，例：fileinput 模块、pathlib 模块等。

一、Python 文件基本操作

  • 文件的基本操作可以分为以下 3 步，每一步都需要借助对应的函数实现：

    • 打开文件：使用 open() 函数，该函数会返回一个文件对象。

    • 对已打开文件做读/写操作：

      • 读取文件，可以使用 read()、readline() 以及 readlines() 函数。

      • 写入文件，可以使用 write() 函数。

    • 关闭文件：完成对文件的读 / 写操作之后，最后需要关闭文件，可以使用 close() 函数。

二、open() 内置函数

  • open() 函数用于创建或打开一个文件，并返回该文件对象。如果该文件无法被打开，会抛出 OSError。

  • open() 函数语法格式：file = open(file_name[, mode='r'][, buffering=-1][, encoding = None])

    • file：表示函数返回的文件对象。

    • file_name：必选参数，用于设定创建 / 打开的文件名，该名称需要使用 单引号 / 双引号 包裹。

      • 如果要打开的文件和当前执行的代码文件位于同一目录，则直接写文件名即可。

      • 如果要打开的文件和当前执行的代码文件不位于同一目录，则需要指定文件所在的完整路径。

    • mode：可选参数，用于设置文件的打开模式，需要使用 单引号 / 双引号 包裹，默认以文本格式（t）、采用只读模式（r）打开文件。

      模式 用法 备注
      r / rt
      （可以不写t）      		 以 文本格式、采用 只读模式 打开文件，读文件内容的指针会放在文件的首位。 操作的文件必须存在。
      rb 以 二进制格式、采用 只读模式 打开文件，一般用于非文本文件，如图片文件、音频文件等。
      r+ / rt+
      （可以不写t）      		 以 文本格式、采用 读写模式 打开文件，读写文件的指针会放在文件的首位，既可以从首位开始读取文件内容，也可以从首位开始向文件中写入新的内容，写入的新内容会覆盖文件中等长度的原有内容。
      rb+ 以 二进制格式、采用 读写模式 打开文件，一般用于非文本文件，如图片文件、音频文件等。
      w / wt
      （可以不写t）      		 以 文本格式、采用 只写模式 打开文件。 若文件存在，会清空文件原有内容；反之，创建新文件。
      wb 以 二进制格式、采用 只写模式 打开文件。
      w+ / wt+
      （可以不写t）      		 以 文本格式、采用 读写模式 打开文件。
      wb+ 以 二进制格式、采用 读写模式 打开文件。
      a / at
      （可以不写t）      		 以 文本格式、采用 追加模式 打开文件。对文件只有写入权限。 若文件存在，会将文件指针放在文件末尾；反之，创建新文件。
      ab 以 二进制格式、采用 追加模式 打开文件。对文件只有写权限。
      a+ / at+
      （可以不写t）      		 以 文本格式、采用 读写模式 打开文件。对文件只有写权限。
      ab+ 以 二进制格式、采用 读写模式 打开文件。对文件只有写权限。

    • buffering：可选参数，用于指定对文件做读写操作时使用缓冲区的大小，建议不做修改。

      • 如果 buffering 参数的值为 0（或者 False），则表示在打开指定文件时不使用缓冲区。

      • 如果 buffering 参数值为大于等于 1 的整数，则表示使用指定的缓冲区大小（单位：字节）。

      • 如果 buffering 参数的值为负数，则表示使用默认的缓冲区大小。

    • encoding：可选参数，用于设定打开文件时所使用的编码格式，一般使用 utf8。

      • 目标文件使用的编码格式和 open() 函数使用的编码格式必须匹配，否则会抛出 UnicodeDecodeError 异常。

三、file 文件对象

  • 通过 open 函数打开文件时会返回 file 文件对象，可以通过文件对象调用自身拥有的属性及方法。

3.1 文件对象常用的属性

  • file.name：返回打开文件的名称。

  • file.mode：返回打开文件时使用的文件打开模式。

  • file.encoding：返回打开文件时使用的编码格式。

  • file.closed：判断文件是否关闭。

3.2 文件对象常用的方法

3.2.1 file.read()

  • 语法格式：file.read([size])

    • size：可选参数。从文件中读取的字符数（文本模式 t）或字节数（二进制模式 b），默认为 -1，表示读取整个文件。

    • 返回值：从文件中读取的内容。

  • 作用：从文件读取指定的字符数（文本模式 t）或字节数（二进制模式 b），如果未给定参数 size 或 size 为负数则读取文件所有内容。

  • 注意事项：必须以可读模式（包括 r、r+、rb、rb+）打开文件，否则会抛出 io.UnsupportedOperation 异常。

3.2.2 file.readline()

  • 语法格式：file.readline([size])

    • size：可选参数。限制读取每一行时最多读取的字符数（文本模式 t）或字节数（二进制模式 b）。

    • 返回值：从文件中读取的一行内容。

  • 作用：和 read() 方法不同，readline() 方法以 “行” 作为读取单位，即每次都读取目标文件中的一行。

    • 对于读取以文本格式打开的文件，读取一行很好理解。

    • 对于读取以二进制格式打开的文件，它们会以 \n 作为读取一行的标志。

  • 注意事项：

    • 必须以可读模式（包括 r、r+、rb、rb+）打开文件，否则会抛出 io.UnsupportedOperation 异常。

    • readline() 方法在读取文件中一行的内容时，会读取最后的换行符 \n，使用 print() 函数输出文件内容时会看到多出了一个空行（print() 函数默认换行）。

3.2.3 file.readlines()

  • 语法格式：file.readlines()

    • 返回值：文件每行内容形成的字符串列表。

  • 作用：readlines() 方法用于读取文件中的所有行（直到结束符 EOF），它和不指定 size 参数的 read() 方法类似，只不过该方法返回是一个字符串列表，列表中的每个元素为文件中的一行内容。

  • 注意事项：

    • 必须以可读模式（包括 r、r+、rb、rb+）打开文件，否则会抛出 io.UnsupportedOperation 异常。

    • readlines() 方法和 readline() 方法相同，在读取每一行时，会连同行尾的换行符一起读取。

3.2.4 file.write()

  • 语法格式：file.write(string)

    • string：写入文件的字符串（文本模式 t）或字节串（二进制模式 b）。

    • 返回值：写入的内容长度。

  • 作用：向文件中写入字符串（文本模式 t）或字节串（二进制模式 b）。

  • 注意事项：

    • 必须以 r+、w、w+、a 或 a+ 的模式打开文件，否则会抛出 io.UnsupportedOperation 异常。

    • 在文件关闭（close()）或缓冲区刷新（flush()）之前，字符串内容存储在缓冲区中，这时在文件中看不到新写入的内容。

    • 如果以文本格式打开文件，那么写入字节串时，要使用字节串的 decode 方法将字节串转换为文本形式，否则报错：TypeError: write() argument must be str, not bytes。

    • 如果以二进制格式打开文件，那么写入字符串时，要使用字符串的 encode 方法将字符串转换为二进制形式，否则报错：TypeError: a bytes-like object is required, not 'str'。

    • 对于以二进制格式打开的文件，可以不使用缓冲区（buffering 为 0），写入的数据会直接进入磁盘文件；但对于以文本格式打开的文件，必须使用缓冲区，否则会抛出 ValueError 异常。

3.2.5 file.writelines()

  • 语法格式：file.writelines([strSeq])

    • strSeq：写入文件的字符串（文本模式 t）或字节串（二进制模式 b）序列。

    • 返回值：无。

  • 作用：向文件中写入字符串（文本模式 t）或字节串（二进制模式 b）列表。

  • 注意事项：write() 方法要注意的点在 writelines() 方法中同样需要注意。

    • 使用 writelines() 方法向文件中写入序列时，不会自动为序列中的各元素添加换行符。

3.2.6 file.tell()

  • 语法格式：file.tell()

    • 返回值：文件指针所在位置。

  • 作用：返回文件的当前位置，即文件指针当前位置。

3.2.7 file.seek()

  • 语法格式：file.seek(offset[, whence])

    • whence：可选参数。用于指定文件指针要放置的位置，该参数的参数值有 3 个选择：0 代表文件头（默认值）、1 代表当前位置、2 代表文件尾。

    • offset：表示相对于 whence 位置文件指针的偏移量，正数表示向后偏移，负数表示向前偏移。

    • 返回值：如果操作成功，则返回新的文件位置，如果操作失败，则函数返回 -1。

  • 作用：将文件指针移动至指定位置。

  • 注意事项：

    • Pyhon3 在没有使用二进制模式（b）打开文件时，只允许从文件头开始计算相对位置，从文件尾计算时会抛出 io.UnsupportedOperation 异常。

3.2.8 file.truncate()

  • 语法格式：file.truncate([size])

    • size：可选参数。用于指定从文件头开始要截断的字节数。
    • 返回值：截断的字节长度。

  • 作用：用于从文件头开始截断文件并返回截断的字节长度。

  • 注意事项：

    • 必须以 r+、w、w+、a 或 a+ 的模式打开文件，否则会抛出 io.UnsupportedOperation 异常。

3.2.9 file.close()

  • 语法格式：file.close()

    • 返回值：无。

  • 作用：用于关闭一个已打开的文件。

  • 注意事项：

    • close() 方法允许被调用多次。

    • 关闭后的文件不能再进行读写操作， 否则会抛出 ValueError 异常。

    • 当 file 对象指向另一个文件时，Python 会自动关闭之前的 file 对象。

3.2.10 file.flush()

  • 语法格式：file.flush()

    • 返回值：无。

  • 作用：刷新缓冲区，即将缓冲区中的数据立刻写入文件，同时清空缓冲区。

    • 注：file.close() 会自动刷新缓冲区。

四、with 用法及原理

4.1 上下文管理器

  • 上下文即执行当前代码所需的相关环境。

  • 上下文管理器顾名思义是用于管理上下文（即环境）的，模式为 “上文 - 操作 - 下文”，即执行代码之前做一些预处理工作，执行之后做一些清理工作。

  • 上下文管理器的常见使用场景：

    • 打开一个文件，然后进行操作，操作完成后关闭文件。

    • 连接一个数据库，然后进行操作，操作完成后关闭连接。

    • 连接一台服务器，然后获取数据上传数据，操作完成后关闭连接。

  • Python 引入 with as 语句，让我们用简单的代码实现上下文的管理，省去 try / finally 的繁琐操作，让代码简化。同时，with as 语句解决了异常发生时，资源无法及时被释放的问题。

4.2 魔法函数

  • 以双下划线（__）开头和结尾的函数被称为魔法函数，它们在进行特定操作时会被自动调用。

  • 魔法函数不是通过继承等机制获得的，而是类定义时 Python 自动赋予给类的特殊函数。开发者不能自己创建魔法函数（可以重写），即使函数名以双下划线开头和结尾。通过魔法函数可以实现许多个性化、便捷的操作。

  • 常用魔法函数：

    • 构造与初始化：__new__ 和 __init__

    • 字符串表示：__str__ 和 __repr__

    • 容器类操作：__len__、__getitem__、__setitem__、__delitem__ 和 __contains__

    • 迭代相关：__iter__ 和 __next__

    • 可调用：__call__

    • with 上下文管理器：__enter__ 和 __exit__

    • 数值转换：__abs__、__bool__、__int__、__float__、__hash__ 和 __index__

    • 元类相关：__new__ 和 __init__

    • 属性相关：__getattr__、__setattr__、__getattribute__、__setattribute__ 和 __dir__

    • 属性描述符：__get__、__set__ 和 __delete__

    • 协程：__await__、__aiter__、__anext__、__aenter__ 和 __aexit__

    • 一元运算符：__neg__（-）、__pos__（+） 和 __abs__

    • 二元运算符：__lt__（<）、__le__（<=）、__eq__（==）、__ne__（!=）、__gt__（>）和 __ge__（>=）

    • 算术运算符：__add__（+）、__sub__（-）、__mul__（*）、__truediv__（/）、__floordiv__（//）、__mod__（%）、__divmod__ 或 divmod()、__pow__ 或 pow()（**）和 __round__ 或round()

    • 反向算术运算符：__radd__、__rsub__、__rmul__、__rtruediv__、__rfloordiv__、__rmod__、__rdivmod__ 和 __rpow__

    • 增量赋值算术运算符：__iadd__、__isub__、__imul__、__ifloordiv__ 和 __ipow__

    • 位运算符：__invert__（~）、__lshift__（<<）、__rshift__（>>）、__and__（&）、__or__（|）和 __xor__（^）

    • 反向位运算符：__rlshift__、__rrshift__、__iand__、__ixor__ 和 __ior__

    • 增量赋值运算符：__ilshift__、__irshift__、__iand__、__ixor__ 和 __ior__

    • 其他魔法函数：__ unicode__()，__ delattr__()， __ del__()， __dict__()，__all__()

4.3 with as 语句

  • with as 语句适用于对资源进行访问的场合，确保不管使用过程中是否发生异常都会执行必要的 “清理” 操作，释放资源，例：文件使用后自动关闭、线程中锁的自动获取和释放等。

  • with as 语句要求 with 后面的 expression 实现了 “上下文管理协议”，即实现了下面两个方法。实现这两个方法的类便是上下文管理器。

    • __enter__(self)：执行 with 后面的 expression时，上下文管理类中的 __enter__() 方法被调用，通常返回一个实例化对象。

    • __exit__(self, exc_type, exc_valye, traceback)：当 with 后面的代码块全部执行完毕后，将调用前面返回的实例化对象的 __exit__() 方法，一般用作异常处理。

  • Python 内置了一些支持上下文管理协议的对象：

    • file

    • decimal.Context

    • thread.LockType

    • threading.Lock

    • threading.RLock

    • threading.Semaphore

    • threading.BoundedSemaphore

  • with as 语句的执行过程如下：

    • 执行上下文表达式（expression）以获得上下文管理器。

    • 载入上下文管理器的 __exit__() 方法以便后续使用。

    • 载入上下文管理器的 __enter__() 方法以便后续使用。

    • 调用上下文管理器的 __enter__() 方法。如果指定了 [as variable] 说明符，将 __enter__() 的返回值赋给 variable。

    • 执行 with 内的代码块。

    • 调用上下文管理器的 __exit__() 方法。 如果代码块的退出是由异常导致的，那么该异常的 type（异常类型）、value（异常信息）和 traceback（堆栈）会作为参数传给 __exit__() 方法，否则传三个 None。如果 __exit__() 函数内部引发了异常，则会覆盖掉代码块的中的异常。__exit_() 方法返回 True 或 False，表示异常有没有被处理。

      • 如果代码块的退出是由异常导致的，并且来自 __exit__() 方法的返回值为 False（默认），则该异常会被重新抛出，中断代码块及代码块之后语句执行。

      • 如果代码块的退出是由异常导致的，并且来自 __exit__() 方法的返回值为 True，则该异常会被忽略，停止执行代码块的剩余语句，但是会继续执行代码块后面的语句。

五、常用文件模块

5.1 pickle 模块：实现 Python 对象的持久化存储

  • pickle 模块能够实现任意对象与文本之间的相互转化，也可以实现任意对象与二进制之间的相互转化。也就是说，pickle 可以实现 Python 对象的存储及恢复。

  • pickle 模块提供了 dumps()、loads()、dump() 和 load() 这 4 个函数供我们使用。

    • dumps 和 loads 函数实现了基于内存的 Python 对象与二进制之间的相互转化。

    • dump 和 loads 函数实现了基于文件的 Python 对象与二进制之间的相互转化。

5.2 os 模块：实现 Python 程序与操作系统进行交互

  • os 模块中大多数接受路径作为参数的函数也可以接受 “文件描述符” 作为参数。

    • 文件描述符：file descriptor，在 Python 文档中简记为 fd，是一个与某个打开的文件对象绑定的非负整数，打开已有文件或新建文件时，系统内核会返回一个文件描述符。

5.2.1 常用功能

（1）属性

  • os.name 属性返回当前操作系统的类型，返回值 posix，nt，java 分别对应 linux，windows，java虚拟机。

  • os.environ 属性返回 字典类型 的环境变量。

（2）函数

  • os.walk(top[, topdown=True[, onerror=None[, followlinks=False]]])：主要用来遍历以 top 为根节点目录内的各个子目录和子文件。返回值是一个生成器。

    • 参数：

      • top：要遍历的目录。

      • topdown：可选参数。当 topdown=True 时，优先遍历 top 目录，然后遍历子目录和子文件；当 topdown=False 时，优先遍历子目录和子文件，然后遍历 top 目录。

      • onerror：可选参数。onerror 是一个函数，当 walk 函数抛出异常时它会被自动调用，参数是一个 OSError 实例。

      • followlinks：可选参数，默认为 False。当 followlinks=True 时，则会遍历目录下的快捷方式（linux 下是 symbolic link）实际指向的目录。
