一、面向对象特征之三：多态性（Polymorphism）

1.1 理解多态性

  • “多态性” 一词最早用于 生物学，指同一种族的生物体具有不同的特性。例：猫科动物中的猫、狮子、老虎和豹等的叫声不同，习性不同。

  • 多态是面向对象程序设计的重要特征之一，是扩展性在 “继承” 之后的又一重大表现。

  • Java 多态性是指在父类中定义的属性和方法被不同的子类继承后，子类可以分别重写父类的方法，让他们方法的表现形式不同，即：多态是同一个行为具有多个不同表现形式或形态的能力。

  • Java 多态性包含 编译时的多态性、运行时的多态性 两大类。

    • 编译时多态又称为静态多态，主要是指方法的 重载。

    • 运行时多态又称为动态多态或动态方法调度，它是一种在运行时对 重写方法 调用的机制。

      • 通过动态绑定实现运行时多态，使用 父类引用指向子类对象，这也是 大家通常说的多态性。

      • 动态绑定：在运行时判断所引用对象的实际类型，根据其实际的类型调用其相应的方法。

1.2 多态性的使用

  • Java 对象的引用变量有两个类型：编译时类型 和 运行时类型。

    • 编译时类型由声明该变量时使用的类型决定，运行时类型由实际赋给该变量的对象决定。简称：编译时看左边，运行时看右边。

    • 若编译时类型和运行时类型不一致，就出现了对象的多态性。

    • 当出现对象的多态性时：看左边 看的是 父类的引用（编译时只能调用父类与子类共有的方法），看右边 看的是 子类的对象（运行时实际调用的是子类重写父类的方法）。

  • 虚拟方法调用：当调用子父类同名同参数的方法时，实际执行的是子类重写父类的方法。

    • 子类中定义了与父类同名同参数的方法，在多态情况下，将此时父类的方法称为虚拟方法（虚方法），父类根据赋给它的不同子类对象，动态调用属于子类的重写方法，这样的方法调用在编译时是无法确定的。

  • Java 的多态性只适用于成员方法，不适用于属性（编译和运行都看左边），因为属性不能被重写。

  • Java 实现多态有 3 个必要条件：继承、重写和向上转型。

    • 继承：在多态中必须存在有继承关系的子类和父类。

    • 重写：子类对父类中某些方法进行重新定义，在调用这些方法时就会调用子类的方法。

    • 向上转型：父类引用指向子类对象。格式：父类类型 对象名 = new 子类类型();

  • 若子类重写了父类的 成员方法，就意味着子类里定义的方法彻底覆盖了父类里的同名方法，系统将不可能把父类里的方法转移到子类中（编译看左边，运行看右边）。对于 成员属性 则不存在这样的现象，即使子类里定义了与父类完全相同的成员属性，子类中的成员属性也不可能覆盖父类中的成员属性（编译运行都看左边）。

1.3 向下转型

  • 使用了对象的多态性后，内存中实际上是加载了子类特有的属性和方法的，但是由于对象声明为父类类型，导致编译时，对象只能调用父类中声明的属性和方法，子类中特有的属性和方法不能被调用。

  • 如果想要调用子类特有的属性和方法，我们可以通过 向下转型，即使用强制类型转换符将父类对象强制转为子类对象。

  • 使用向下转型时可能会出现 java.lang.ClassCastException 的异常。

  • 可以通过 instanceof 关键字来判断是否可以使用向下转型。

  • 问题：编译时通过，运行时不通过。

  • 面试题、

    • 判断 base.overWrite(1, 2, 3); 调用的方法及输出。

      • base.overWrite(1, 2, 3); 调用的是子类重写的方法。如何调用子类重载的方法？

1.4 instanceof 关键字

  • 在 Java 中可以使用 instanceof 关键字判断一个对象是否为一个类（或接口、抽象类、父类）的实例。

    • 语法格式：boolean result = obj instanceof Class，其中，obj 是一个对象，Class 表示一个类或接口。

    • 当 obj 是 class 类（或接口、子类）实例时，result 返回 true，否则返回 false。
