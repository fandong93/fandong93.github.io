1.6 配置虚拟环境

  • 默认情况下，conda 创建的虚拟环境及安装的模块缓存都存储在用户目录 C:\Users\username\.conda\envs 下，这一点不会在 conda 配置文件 .condarc 中体现出来。当前的默认信息可通过 conda info 指令打印查看，其中不仅能够查看当前默认环境路径和默认缓存路径，还能查看 conda 的源设置。

    • 修改 .condarc 中的 env_dirs 和 pkgs_dirs 配置项，可以分别设置虚拟环境和缓存路径，配置项中的第一个路径会作为默认路径。

  • 创建虚拟环境：conda create --name <env_name> [--prefix <path>] [interpreter_version] [package_names]

    • --name 参数指定虚拟环境名称，缩写为 -n。

    • <env_name>：创建的虚拟环境名称。

    • --prefix 参数指定虚拟环境路径，缩写为 -p。

    • <path>：创建的虚拟环境路径。

    • [interpreter_version]：创建虚拟环境时指定的 Python 解释器版本，可选参数。例：python=3.9。

    • [package_names]：创建时需要安装的第三方包，多个之间包用空格隔开。例：requests flask。

      • 所有的包，不论 base 环境还是虚拟环境都是放在 pkgs 文件夹下。如果虚拟环境需要安装的包与 pkgs 中已有的包版本完全一样，则不会再下载。当一个包被多个环境使用时，从某一个环境卸载该包也不会将其从 pkgs 文件夹删除。

    • 如果报错 NoWritablePkgsDirError: No writeable pkgs directories configured.，需将 pkgs_dirs 路径权限进行修改，允许用户写入。

  • 创建与 base 环境基本相同的虚拟环境：conda create -n <env_name> [--prefix <path>] [interpreter_version] anaconda

  • 查看虚拟环境：conda env list / conda info --envs

  • 激活虚拟环境：

    • Windows：conda activate env_name

      • 报错 CommandNotFoundError: Your shell has not been properly configured to use 'conda activate' 时，运行 conda init cmd.exe。

    • Linux & Mac：source activate env_name

  • 退出虚拟环境：退出虚拟环境时并不需要指定虚拟环境名。

    • Windows：conda deactivate

    • Linux & Mac：source deactivate

  • 修改虚拟环境名称：

    • 将旧环境克隆到新环境中：conda create --name new_name --clone old_name

    • 删掉旧环境：conda remove --name old_name --all

  • 共享虚拟环境：conda 提供了将虚拟环境中的所有依赖包统一导出到一个配置文件中的功能，在本机之外的机器上使用这套代码时，根据 conda 导出的配置文件重建虚拟环境即可。

    • 导出依赖包到配置文件：conda env export --file environment.yml

      • --file 参数缩写为 -f，可以替换为 >。

      • environment.yml 为导出的文件，其中文件名 environment 可以更改。

      • 如果导出的路径中已经有了 environment.yml 文件，conda 会重写这个文件。

    • 根据配置文件创建虚拟环境：conda env create -f environment.yml

  • 删除虚拟环境：conda remove -n env_name --all

1.7 conda 常用指令

  • 查看 conda 版本：conda --version

  • 更新至最新版本：conda update conda

  • 查找指定包的信息：conda search package_name

  • 更新所有包：conda update --all

  • 更新当前环境下的指定包：conda update package_name

  • 更新指定环境下的指定包：conda update --name env_name package_name

  • 查看所有已安装的包：conda list

  • 查看指定环境所有已安装的包：conda list --name env_name

  • 在当前环境中安装包：conda install package_name

  • 在指定环境中安装包：conda install --name env_name package_name

  • 删除当前环境中的包：conda remove package_name

  • 删除指定环境中的包：conda remove --name env_name package_name

1.8 PyCharm 使用 Jupyter

  • 生成 Jupyter 配置文件：jupyter notebook --generate-config

    • Writing default config to: C:\Users\641743220\.jupyter\jupyter_notebook_config.py

  • 设置密码：jupyter notebook password（输入密码、验证密码）

    • Wrote hashed password to C:\Users\641743220\.jupyter\jupyter_notebook_config.json

  • 设置服务器启动时打开的文件夹（默认文件夹 C:\Users\username）

    • 打开生成的配置文件 jupyter_notebook_config.py，设置 c.NotebookApp.notebook_dir = F:/，要将其中的 \ 转为 /。

  • 登录服务器：jupyter notebook

二、CUDA 11

2.1 下载 NVIDIA 驱动

  • 根据 PyTorch 版本查看可安装的 cuda 版本：https://pytorch.org/get-started/locally/

  • 查看 cuda 与 NVIDIA 驱动版本的对照：https://docs.nvidia.com/cuda/cuda-toolkit-release-notes/index.html

  • 官网 https://www.nvidia.cn/geforce/drivers/ 选择自己电脑显卡的型号（注意：笔记本的话选择型号后面带 Notebooks 的），下载最新的驱动安装包，双击安装即可。

2.2 查看 NVIDIA 驱动

  • 使用 nvidia-smi 查看 NVIDIA 驱动信息：nvidia-smi 是 NVIDIA 的系统管理界面 ，其中 smi 是 system management interface 的缩写，它可以收集各种级别的信息，查看显存使用情况。

  • 解决报错：'nvidia-smi' 不是内部或外部命令，也不是可运行的程序或批处理文件。

    • 找到 nvidia-smi.exe 程序所在目录。

    • 将路径添加到 path 系统环境变量。

2.3 安装 CUDA 工具包

  • 官网下载适合自己电脑 NVIDIA 驱动的 CUDA 工具包：https://developer.nvidia.com/cuda-toolkit-archive

  • 双击安装

  • 验证安装

三、cudnn

3.1 下载 cudnn

  • 进入 cudnn 官网：https://developer.nvidia.com/zh-cn/cudnn 下载 cuDNN 安装包。注：需要使用 NVIDIA 账号登陆网站才可以下载，没有账号注册即可。

3.2 安装 cudnn

  • 解压压缩包 cudnn-windows-x86_64-8.6.0.163_cuda11-archive.zip，解压后有如下文件及文件夹。

  • 将 bin、include、lib 三个文件夹 合并 到 CUDA 安装目录：C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.7。

  • 编辑 Path 环境变量，加入下列四条路径。

    • C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.7\bin

    • C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.7\include

    • C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.7\lib\x64

    • C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.7\libnvvp

  • 测试 cudnn 是否安装成功

    • cmd 进入到 C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.7\extras\demo_suite 路径下。

    • 分别执行 deviceQuery.exe 和 bandwidthTest.exe。

四、安装 GPU 版本 PyTorch

  • https://download.pytorch.org/whl/torch_stable.html 下载 torch 安装包。GPU 版本以 cu 开头，cuda 11.7 对应于 cu117。cp 后面对应于电脑已安装的 python 版本，anaconda 2022.10 自带的 python 版本为 3.9。因此，下载 torch 安装包：torch-1.13.0+cu117-cp39-cp39-win_amd64.whl。

  • 同理下载 torchvision 安装包：cu117/torchvision-0.14.0%2Bcu117-cp39-cp39-win_amd64.whl。

  • 进入到 torch 和 torchvision 的下载位置进行安装。

    • pip install "torch-1.13.0+cu117-cp39-cp39-win_amd64.whl"

    • pip install "torchvision-0.14.0+cu117-cp39-cp39-win_amd64.whl"

  • 检查是否安装成功：

    • conda list / pip list

    • 导入 torch
